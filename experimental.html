Experimental folder in machop

This folder started as a precursor for the Unibuddy design system (DS). It quickly became the place for all the new low level components. There is an important distinction we need to make in machop which is at what level of complexity/reusability/customisability of components do we keep in Experimental/DS and what do we keep in machop. One way to look at it is, DS project is something we could theoretically open source.

What belongs in Experimental

Going forward we should implement a strategy of separating components. Components that satisfy any of these rules belong in Machop:

  1. If the component is a template or a page
  2. if the component is a collection of other DS components
  3. if the component uses a specific apollo query or mutation
  4. If the component implements product(s) specific business logic

When adding components to Experimental keep in mind:

  • This component should be reusable across products
  • Therefore, colours, sizing and other design factors should be customisable, theme-able and work across different contexts
  • It should be extensible through composition, see how LoaderButton composes over Button. LoaderButton passes all the props down, only adds loading functionality, but also is aware of how Button is styled (and in early versions introduced visual regressions because the css of the two components clashed)
  • component should be well documented

Theming

  • single design system
  • using css variables for theming
# in button.pcss .button {
    height: var(--ds-button--height-md);
}
.button.primaryColor {
    background-color: var(--ds-button--bg-color-primary);
}
# in onix/theme.pcss :root {
    --ds-button--height-md: 32px;
    --ds-button--bg-color-primary: #ccc;
}
# in abra/theme.pcss :root {
    --ds-button--height-md: 42px;
    --ds-button--bg-color-primary: #f9f9f9;
}
  • can be configured per app or product
  • base components should be the same, Button should be reusable across products
  • as a last resort creating product-local low level components

Migrating Experimental

  • at some point we will need to swap all the imports of @unibuddy/machop/Experimental with @unibuddy/design_system

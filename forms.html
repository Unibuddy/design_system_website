Forms

Get started

Our form components are a thin wrapper around Formik and yup for validation. To start wrap all your form inputs in a <Form> component.

<Form>
    <input value={this.state.value} onChange={this.handleChange} />
    <button type="submit">Submit</button>
</Form>

Next we need to wrap the input in a <FormGroup> component in order to wire up the input with the <Form>. Next we can remove the value and onChange props as we don't need to control Formik components, the <Form> holds all the state. Our page only declares what inputs it needs. label="Test" adds a label element and associates it with the input via a generated or manually provided id.

<Form>
    <FormGroup label="Test">
        <input name="test" />
    </FormGroup>
    <button type="submit">Submit</button>
</Form>

In order for formik to differentiate inputs we need to add a name to each of them. See name="test" First, we will replace input with TextInput for some nice default styling. We can replace the button with a <SubmitButton> that abstracts away the disabling of the button and loading state.

<Form>
    <FormGroup label="Test">
        <TextInput name="test" />
    </FormGroup>
    <SubmitButton>Submit</SubmitButton>
</Form>

Example

Adding validation

Finally to validate the form we add validationSchema and initialValues (that way formik knows which fields are invalid before any onChange is fired).

const initialValues = {test: ""};
const validationSchema = {
    test: yup
        .string()
        .required()
        .min(3),
};
<Form validationSchema={validationSchema} initialValues={initialValues}>
    <FormGroup label="Test">
        <input name="test" />
    </FormGroup>
    <SubmitButton>Submit</SubmitButton>
</Form>;

onSubmit

In order to get the data from the form we must add a onSubmit callback to the Form component. On form submit Formik asynchronously validates the form and if all the validation passes calls our onSubmit callback with values as the first argument and internal actions as the second arg.

handleSubmit(values, formikActions) {
 try {
   await api(...);
   # after successuful api call
   formikActions.setSubmitting(false);
 } catch (error) {
   formikActions.setSubmitting(false);
   formikActions.setValidation("serverError", error.message);
 }
}
<Form onSubmit={handleSubmit}>
    {...}
    <ErrorMessage name="serverError" />
</Form>

Adding a custom form input

In order to add a new input type that interfaces with <FormGroup and <Form> it needs to implement a kind of interface. The input needs to be controlled, meaning it has to accept a value and name props and emit props.onChange and props.onBlur that emit a "react like" event. Example datepicker where <DatePicker/> is some random library that has weird props like onSelect and inputProps etc...:

export default function DatepickerInput(props) {
    return (
        <DatePicker
            selectedDate={props.value}
            inputProps={{onBlur: props.onBlur}}
            onSelect={date =>
                props.onChange({
                    target: {value: date, name: props.name},
                })
            }
        />
    );
}
